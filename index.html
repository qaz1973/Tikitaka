基础设置

从预设、键位、行程、RT、校准、测试到保存，按真实配置顺序整理。

设备预设

按设备型号套用基础配置，适合作为第一次配置或恢复标准配置的起点。

设备型号快速套用

操作流程

  1. 进入磁轴配置页面，打开“设备预设”。
  2. 选择与你设备型号一致的预设。
  3. 确认套用后检查按键数量、画布位置、通道和灯光是否符合当前设备。
  4. 确认无误后保存配置。

完成后检查

  • 按键数量和实际面板一致。
  • 设备预设没有把不需要的 GPIO、LED 或外设配置写入当前设备。

注意事项

  • 设备预设可能同时影响磁轴通道、GPIO、LED、外设等硬件段，套用前建议确认当前配置是否需要备份。

依据：HallEffectConfigPage.tsx、www/public/presets

布局预设

根据实际面板布局快速生成按键位置和通道关系。

布局通道关系

操作流程

  1. 进入磁轴配置页面，打开“布局预设”。
  2. 选择与外壳或键位排列一致的布局。
  3. 套用后检查每个按钮在画布上的位置和通道编号。
  4. 如果只需要调整少量键位，再进入后续按键功能或通道分配步骤。

完成后检查

  • 方向键、功能键和辅助键位置清楚可辨。
  • 布局不会覆盖你已经调整好的自定义位置。

注意事项

  • 布局预设主要用于画布、按钮和动作关系；如果硬件通道也要跟着改变，请再检查通道分配。

依据：HallEffectConfigPage.tsx、www/public/presets/models

按键功能调整

点击目标按键并修改它对应的 GP2040 动作。

按键功能动作映射

操作流程

  1. 在磁轴配置画布上选择一个按键。
  2. 在右侧属性面板中选择“功能”。
  3. 按需要改成方向、B1/B2、L1/R1、S1/S2、A1/A2、Fn 等动作。
  4. 逐个确认关键按键后保存。

完成后检查

  • 画布上的标签和你的目标键位一致。
  • 进入测试模式后按下按键，触发状态和显示的动作一致。

注意事项

  • HE 通道和 GPIO 映射都可以分配功能，但 GPIO 按键没有行程和 RT 参数。

依据：HallEffectConfigPage.tsx、proto/enums.proto

单键行程触发设置

只调整一个磁轴按键的总行程和触发行程，适合微调单个键的手感。

TravelActuation单键

操作流程

  1. 选择一个 HE 按键。
  2. 切换到“行程”工具。
  3. 设置总行程 Travel，通常按实际轴体行程填写。
  4. 设置触发行程 Actuation，数值不能大于总行程。
  5. 保存前先进入测试模式确认触发位置。

完成后检查

  • 轻按到目标深度时按键开始触发。
  • 完全释放后不会残留触发状态。

注意事项

  • 触发行程越小越早触发，但过小会增加误触风险。
  • GPIO 按键不支持行程设置。

依据：HallEffectConfigPage.tsx、proto/config.proto

批量行程触发设置

同时选择多个磁轴按键并统一应用 Travel 和 Actuation 参数。

Travel批量

操作流程

  1. 按住多选或框选需要统一手感的按键。
  2. 切换到“行程”工具。
  3. 输入总行程和触发行程。
  4. 批量应用后逐个测试常用方向键或攻击键。
  5. 确认无误后保存。

完成后检查

  • 同组按键触发手感一致。
  • 没有把不需要调整的按键一起选中。

注意事项

  • 批量设置会覆盖所选按键原有参数，适合 WASD、方向键或同规格按键统一设置。

依据：HallEffectConfigPage.tsx

单键 RT 设置

为单个磁轴按键开启 Rapid Trigger，并设置按下和抬起灵敏度。

Rapid TriggerRT DownRT Up

操作流程

  1. 选择一个 HE 按键。
  2. 切换到“RT”工具。
  3. 打开 Enable Rapid Trigger。
  4. 设置 RT Down 和 RT Up。
  5. 进入测试模式，连续轻按和释放，确认不会误触或粘连。

完成后检查

  • 按下时响应足够快。
  • 轻微抖动时不会反复误触。

注意事项

  • RT 数值越小越灵敏，也越依赖轴体和传感器稳定性。
  • 如果出现误触，先适当增大 RT Down 或 RT Up，再重新测试。

依据：HallEffectConfigPage.tsx、src/webconfig.cpp

批量 RT 设置

对多个磁轴按键统一应用 Rapid Trigger 参数。

Rapid Trigger批量

操作流程

  1. 选择需要统一 RT 手感的多个 HE 按键。
  2. 切换到“RT”工具并开启 Rapid Trigger。
  3. 填写 RT Down 和 RT Up。
  4. 批量应用后在测试模式中逐个按键检查。
  5. 确认后保存配置。

完成后检查

  • 同组按键的触发和复位速度一致。
  • 没有把辅助键或不需要 RT 的按键一起应用。

注意事项

  • 批量 RT 适合方向键和常用动作键，不建议在未测试前直接套到所有按键。

依据：HallEffectConfigPage.tsx

按键校准

校准磁轴按键的空闲点和最大按下点，让行程、触发和 RT 判断有可靠基准。

磁轴校准空闲点按下点

操作流程

  1. 进入按键校准或批量校准入口。
  2. 保持所有按键松开，等待空闲采样完成。
  3. 按界面提示逐个按下目标按键到最大行程。
  4. 等待系统记录按下点后松开。
  5. 全部完成后检查状态，再进入保存校准结果。

完成后检查

  • 空闲状态下按键不触发。
  • 完全按下时状态稳定触发。
  • 测试画面中的行程变化连续，不出现明显跳动。

注意事项

  • 校准时不要压着其他按键。
  • 如果某个键状态异常，重新校准该键或先检查通道/引脚分配。

依据：HallEffectConfigPage.tsx、WebApi.js、src/webconfig.cpp

保存校准结果

校准完成后把新的磁轴参数写入设备，避免刷新或断电后丢失。

保存校准结果

操作流程

  1. 确认所有需要校准的按键已经完成。
  2. 点击保存或保存校准结果。
  3. 等待保存完成提示。
  4. 保存后再做一次 RT 与行程测试。

完成后检查

  • 页面提示保存成功。
  • 刷新页面或重新进入后，校准状态仍然正常。

注意事项

  • 未保存就离开页面、刷新或重启，校准结果可能不会写入设备。

依据：HallEffectConfigPage.tsx、src/webconfig.cpp

RT 与行程测试

在测试状态下检查按键状态、行程读数和 Rapid Trigger 响应是否正常。

状态测试行程RT

操作流程

  1. 打开测试模式。
  2. 逐个按下需要检查的按键。
  3. 观察触发状态、行程读数和释放响应。
  4. 发现异常时返回校准、行程或 RT 设置重新调整。
  5. 测试通过后保存最终配置。

完成后检查

  • 按下、保持、释放三个阶段都稳定。
  • RT 开启的按键能随释放和再按快速响应。

注意事项

  • 测试模式读取运行时状态，不等于已经保存；参数确认后仍要执行保存。

依据：HallEffectConfigPage.tsx、WebApi.js

保存配置到设备

把当前页面的设置写入设备存储。

写入设备重启前确认

操作流程

  1. 确认预设、按键、行程、RT、SOCD 或模式设置都已经检查完。
  2. 点击保存配置。
  3. 等待保存完成提示。
  4. 如果页面提示需要重启，再从顶部导航选择重启方式。

完成后检查

  • 页面出现保存成功或进度完成提示。
  • 重新进入页面后关键参数仍然保留。

注意事项

  • 保存、重启、恢复默认都会改变设备状态，请确认后再执行。
  • 重启到 Controller 模式后可能无法继续打开 Web Config，需要重新进入配置模式。

依据：Navigation.jsx、WebApi.js

控制器模式切换

切换设备启动后的输入模式，例如 XInput、Switch、DInput、Keyboard、PS4/PS5、Xbox 等。

Input Mode控制器模式

操作流程

  1. 进入设置或磁轴页顶部的输入模式入口。
  2. 选择目标控制器模式。
  3. 如该模式需要认证或 USB Host，先完成对应外设配置。
  4. 保存设置。
  5. 必要时重启到 Controller 模式，让系统重新识别设备。

完成后检查

  • 电脑或主机识别出的设备类型符合目标模式。
  • 按键测试工具中输入正常。

注意事项

  • 某些主机模式需要认证文件、认证手柄或 USB Host。
  • 高级 USB 描述符只适合清楚风险时修改，错误设置可能导致模式不可用。

依据：SettingsPage.jsx、HallEffectConfigPage.tsx

SOCD 模式切换

设置相反方向同时输入时的清理规则。

SOCD方向清理

操作流程

  1. 进入磁轴配置页或设置页。
  2. 找到 SOCD 清理模式。
  3. 按需要选择 Up Priority、Neutral、Last Win、First Win 或 Off。
  4. 保存配置并测试左右、上下同时输入。

完成后检查

  • 同时按下相反方向时输出符合你选择的规则。
  • 目标游戏或比赛规则允许当前 SOCD 模式。

注意事项

  • Off/Bypass 表示不清理相反方向输入，合规性需要自行确认。
  • 部分平台或输入模式可能不支持 SOCD Off，会回到 Neutral。

依据：SettingsPage.jsx、proto/enums.proto

进阶设置

专业模式、布局导入、通道/引脚、OLED、灯光和自动校准等高级流程。

切换用户模式

切换普通/专业界面，显示更多布局、灯光、导入导出和高级维护工具。

Pro 模式界面模式

操作流程

  1. 在磁轴配置页面打开“更多”或界面模式入口。
  2. 选择普通模式或 Pro/专业模式。
  3. 进入专业模式后再进行布局解锁、添加按键、灯光工具或自动校准表操作。
  4. 完成后可切回普通模式，减少误操作。

完成后检查

  • 专业模式下可以看到解锁、添加按键、OLED 布局、LED 工具等入口。
  • 普通模式下危险或复杂入口被隐藏。

注意事项

  • 专业模式会暴露更多会改变布局和硬件映射的功能，建议只在需要时打开。

依据：InterfaceMode.ts、HallEffectConfigPage.tsx

导入用户布局

导入自定义布局中的部分配置，用于复用已整理好的按键位置或配置片段。

导入自定义布局

操作流程

  1. 进入专业模式。
  2. 打开导入入口，选择布局或配置文件。
  3. 只导入需要的部分，避免覆盖无关设置。
  4. 导入后检查按键位置、通道、功能和灯光是否正常。
  5. 确认后保存。

完成后检查

  • 导入后的布局和设备实际面板一致。
  • 没有把旧设备的通道或 GPIO 配置误带入当前设备。

注意事项

  • 跨设备导入前建议先备份当前配置。

依据：HallEffectConfigPage.tsx

按键通道与引脚分配

为按键分配磁轴通道或 GPIO 引脚，解决硬件线路和画布按钮之间的对应关系。

HE 通道GPIO引脚

操作流程

  1. 进入专业模式并选择目标按键。
  2. 在属性面板中确认按键类型是 HE 通道还是 GPIO 映射。
  3. 为 HE 按键选择通道编号，或为 GPIO 按键选择对应引脚。
  4. 分配动作功能。
  5. 保存前用测试或捕获功能确认实际按键。

完成后检查

  • 按下实体按键时，画布上对应按钮被触发。
  • GPIO 引脚没有覆盖插件或外设正在占用的引脚。

注意事项

  • 常规 GPIO 映射页支持 Pin Viewer 和多 Profile；磁轴页适合处理 HE 通道和画布布局。
  • 不要随意启用“编辑插件占用引脚”一类危险选项。

依据：HallEffectConfigPage.tsx、PinMapping.tsx

添加按键

在专业布局中新增需要配置的按键元素。

画布编辑新增按键

操作流程

  1. 切换到专业模式并解锁画布。
  2. 选择添加按键工具和按钮尺寸。
  3. 在画布目标位置点击添加。
  4. 为新按键设置 HE 通道或 GPIO 引脚。
  5. 设置动作功能、行程或灯光后保存。

完成后检查

  • 新增按键位置不遮挡其他元素。
  • 新增按键已经绑定通道/引脚和功能。
  • 测试模式中可以正确触发。

注意事项

  • 只添加画布按钮不等于硬件已经可用，必须完成通道或引脚分配。

依据：HallEffectConfigPage.tsx

屏幕定义与布局保存

修改 OLED/屏幕布局中的按钮元素，绑定通道并保存自定义显示布局。

OLED自定义布局保存

操作流程

  1. 进入 OLED 布局或屏幕定义编辑器。
  2. 解锁布局编辑。
  3. 添加椭圆、方形或按钮元素。
  4. 为元素绑定 HE 通道或 GPIO 动作。
  5. 检查位置、大小和显示层级。
  6. 保存自定义布局，并回到显示配置中选择使用。

完成后检查

  • 屏幕预览中的按钮位置与实际设备相符。
  • 显示配置里已经选中自定义布局。
  • 设备屏幕能看到输入模式、SOCD、Profile 或输入历史等需要的状态项。

注意事项

  • OLED 布局保存后，还需要在显示配置中启用显示并选择对应布局。
  • 显示功能依赖 I2C 屏幕和正确的外设映射。

依据：DisplayLayoutEditor.tsx、DisplayConfig.jsx

按键灯光顺序设置

设置 RGB 按键灯在灯带链路中的顺序，影响流水、渐变和按键灯效对应关系。

LED 顺序RGB 链路

操作流程

  1. 进入 LED 配置或专业模式灯光工具。
  2. 确认 RGB 数据脚、LED 格式、每按钮 LED 数和最大亮度。
  3. 按实际灯带焊接顺序排列按钮 LED。
  4. 保存后观察灯效是否与实体按键顺序一致。

完成后检查

  • 第一个按钮 LED 从索引 0 开始。
  • 灯效移动方向和实际灯带顺序一致。
  • 玩家灯或外壳灯索引没有占用按钮灯区间。

注意事项

  • GP2040-CE 文档提示 RGB Button LEDs 应优先从索引 0 配置。
  • 灯光顺序错了，颜色本身可能正常，但动态效果会错位。

依据：LEDConfigPage.jsx、CustomThemePage.jsx

按键灯光颜色设置

为单个或多个按键设置普通状态和按下状态颜色。

按键灯颜色

操作流程

  1. 进入专业模式并选择灯光工具，或进入自定义主题页面。
  2. 选择目标按键。
  3. 设置 Normal Color 和 Pressed Color。
  4. 需要统一颜色时可多选按键批量设置。
  5. 保存并用灯光预览或实体灯检查效果。

完成后检查

  • 松开和按下时颜色符合预期。
  • 颜色没有因为灯光顺序错误映射到其他按键。

注意事项

  • 如果颜色通道异常，先检查 LED Format，例如 GRB/RGB/GRBW/RGBW。

依据：LEDConfigPage.jsx、CustomThemePage.jsx

氛围灯配置或预设

为外壳或设备内部的 Case RGB/氛围灯添加配置，或套用已有预设。

氛围灯Case RGB预设

操作流程

  1. 进入专业模式或 LED 配置的 Case RGB 区域。
  2. 选择 Off、Ambient 或 Linked 等颜色类型。
  3. 设置氛围灯起始索引和数量。
  4. 选择已有预设，或新增自定义配置。
  5. 保存后观察实体氛围灯。

完成后检查

  • 氛围灯索引位于按钮灯之后，且没有和玩家灯冲突。
  • Ambient 模式下氛围灯可以独立于按钮灯工作。

注意事项

  • 外壳 RGB 的起始索引必须按实际灯带顺序设置。

依据：LEDConfigPage.jsx、HallEffectConfigPage.tsx

氛围灯颜色设置

调整氛围灯的默认颜色或预设颜色。

氛围灯颜色

操作流程

  1. 选择已经添加的氛围灯配置。
  2. 打开颜色选择器。
  3. 设置空闲颜色或预设颜色。
  4. 保存并确认实体灯颜色。

完成后检查

  • 氛围灯颜色改变的是外壳/背景灯，不是按键灯。
  • 颜色和亮度不会影响按键识别或供电稳定性。

注意事项

  • 如果灯光数量较多，亮度过高可能增加供电压力；异常时先降低最大亮度。

依据：LEDConfigPage.jsx

自动校准

开启或关闭磁轴自动校准，让设备根据环境变化学习和应用校准 Profile。

自动校准Profile

操作流程

  1. 进入磁轴配置页面。
  2. 找到自动校准开关。
  3. 按需要开启或关闭。
  4. 开启后先做一次完整手动校准和测试。
  5. 保存配置。

完成后检查

  • 启用后按键状态仍然稳定。
  • 如果环境或设备状态变化后，按键没有出现异常触发。

注意事项

  • 自动校准适合温漂或环境变化导致空闲点偏移的场景。
  • 如果需要完全固定手感，可优先使用手动校准并关闭自动校准。
  • 专业模式中可查看或清空自动校准保存表。

依据：HallEffectConfigPage.tsx、src/addons/he_trigger.cpp
